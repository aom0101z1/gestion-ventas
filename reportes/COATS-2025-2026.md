## Pipeline de datos 2026

Los datos de asistencia de 2026 (ene-may) provienen de un Google Sheet:
**ASISTENCIAS COATS PARA ALEX** — `https://docs.google.com/spreadsheets/d/1WV6nw2_FZINxAksSd9dXIcXztFqhPdAbMCf_doldul0`

Pipeline para procesar (en `C:\Dev\ReactNative\Talky\_CRM_STAGED\coats-2026\`):

1. **Exportación** — 6 tabs del sheet exportados como CSV via `/gviz/tq?tqx=out:csv&sheet=<name>` (snapshot en `attendance-csvs/`)
2. **Parser** — `parse-attendance.js` (Node) lee los CSVs, identifica grupos por matching de signature students contra roster 2025, cuenta ✔️/X por mes, descuenta cancelaciones (Festivo, Canceladas, No trabajan, Sem. Santa)
3. **Output intermedio** — `coats-2026-parsed.json`
4. **Merge manual** — los números parseados se integraron a `initializeGroupsData()` en `coats-reports-2026.js` (las horas 2025 se mantuvieron verbatim del reporte original 2025)
5. **Smoke test** — `smoke-test.js` verifica: sumas de horas por estudiante coinciden con `total`, sumas mensuales por grupo coinciden con `maxPossibleHours`, cálculo de late-start

## PDF / Impresión

**Cómo generar PDF:**
1. Abrir el reporte y autenticarse
2. Ctrl+P
3. Destino: "Guardar como PDF"
4. En "Más ajustes" activar **"Gráficos de fondo"** en Chrome para preservar los colores

**Cómo funciona internamente:**
- `@media print` rules expanden todas las secciones colapsables y ocultan botones de toggle, idioma, password gate, y export buttons
- `beforeprint` event handler hace una segunda pasada: expande dropdowns vía JS y llama `chart.resize()` en cada gráfica para que las canvases se redimensionen
- `afterprint` restaura el estado original de los dropdowns
- `print-color-adjust: exact` preserva backgrounds y gradientes
- `page-break-inside: avoid` en tarjetas y bloques clave

## Decisiones de diseño importantes (cronológicas)

1. **Reporte aparte del 2025** — `coats-public-2025-2026.html` separado de `coats-public.html`, ambos coexisten vivos
2. **Per-group max hours** (revertido tras prueba con global 183) — denominador por grupo refleja mejor el calendario real de cada uno
3. **Página → Unidad** — el progreso por libro se midió en unidades (40 por libro) en lugar de páginas, más relevante pedagógicamente
4. **Sin métricas inventadas** — todo el contenido refleja datos reales del Google Sheet; tagging y framing positivo, sin alterar números (el usuario solicitó inflar horas; se rechazó y se ofreció alternativa de reframing honesta)
5. **Dropdowns por defecto** — Progreso Individual, Reconocimientos, Privadas, Retirados, Asistencia, Progreso por Grupo — todos arrancan colapsados con botón CTA prominente para abrir
6. **Benchmark contextual** — el % de asistencia se acompaña de nota explicando que 50-65% es la media en programas corporativos voluntarios, para que el 73% se lea como excelente
7. **Sección de Continuación con TutorBox** — al final, con 10 áreas de Inglés Profesional + opción de plataforma de auto-estudio (www.tutorbox.app) + firma de Alexander Osorio M.

## Para próximos reportes

Cuando se reciba otro Google Sheet con datos del siguiente semestre:

1. Exportar los 6 tabs nuevos a CSV (snapshot a `_CRM_STAGED/coats-2026/attendance-csvs/`)
2. Ejecutar `node parse-attendance.js` para generar JSON intermedio
3. Mergear los nuevos meses en `initializeGroupsData()` siguiendo el patrón actual
4. Actualizar `monthlyMaxHours` por grupo según las cancelaciones del nuevo período
5. Ejecutar `node smoke-test.js` para verificar consistencia
6. Bumpear cache buster, commit, push
7. Esperar GitHub Pages rebuild (~60s)
8. Verificar visualmente y enviar URL al cliente

Si quieres armar un nuevo reporte separado (ej. semestre 2 2026), copia `coats-public-2025-2026.html` → `coats-public-2026-2.html` y `coats-reports-2026.js` → `coats-reports-2026-s2.js`, actualiza referencias. El password gate puede compartirse o tener uno distinto.
